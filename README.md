## About This Repository

This repository contains the research output of Axon System on the emerging discipline of AI Visibility Engineering — the practice of optimizing brand entity representation across Large Language Model training corpora, Knowledge Graphs, and semantic metadata layers.

All papers are authored by Francesco Tinti (Q139765600) and published under the AIMENSION Protocol framework (Q139783726).

## Papers

### 1. The End of SEO
**File:** `the-end-of-seo.md`  
**Abstract:** Search Engine Optimization optimizes for systems that return ranked lists of URLs. Large Language Models do not return lists. They synthesize answers from statistical patterns. This paper argues that the transition to AI-based information retrieval constitutes a structural discontinuity requiring an entirely new infrastructure discipline: AI Visibility Engineering.  
**Keywords:** SEO, LLM, AI Visibility Engineering, GEO, AEO, Knowledge Graph, entity resolution

### 2. AEO vs. GEO: A Technical Distinction That Changes Everything
**File:** `aeo-vs-geo.md`  
**Abstract:** Answer Engine Optimization and Generative Engine Optimization are technically distinct disciplines operating at different layers of AI architecture — inference-time retrieval versus training-time encoding. Conflating them leads to misallocated intervention strategies. This paper defines the distinction and provides the appropriate technical framework for each.  
**Keywords:** AEO, GEO, Answer Engine Optimization, Generative Engine Optimization, RAG, LLM training, inference, AIMENSION Protocol

### 3. Knowledge Graphs for Enterprise Brands
**File:** `knowledge-graphs-for-brands.md`  
**Abstract:** Complete technical guide to establishing and maintaining Wikidata entity authority for enterprise organizations. Covers entity creation methodology, minimum property sets, cross-source consistency management, and SPARQL-based monitoring. Uses Axon System as a live case study.  
**Keywords:** Wikidata, Knowledge Graph, entity authority, brand AI visibility, entity resolution, SPARQL, Schema.org, JSON-LD
